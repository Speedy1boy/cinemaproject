# Cinema Backend API

Серверная часть приложения, предоставляющая REST API для взаимодействия с клиентским приложением. Отвечает за работу с базой данных, аутентификацию и обработку бизнес-логики.

## Стек технологий
* **Язык:** Java 21
* **Фреймворк:** Spring Boot 3.x (Web, Data JPA, Security)
* **СУБД:** PostgreSQL 15
* **Безопасность:** JWT (JSON Web Tokens)
* **Сборка:** Maven

## Ключевые архитектурные решения
* Реализована защита от "гонки данных" (Race Condition) при параллельном бронировании билетов с использованием транзакций БД.
* Пароли пользователей не хранятся в открытом виде (используется алгоритм хеширования BCrypt).
* Настроена CORS-политика для корректного взаимодействия с фронтендом.

## Локальный запуск (для разработки)

1. Убедитесь, что у вас запущен локальный сервер PostgreSQL.
2. Создайте пустую базу данных `cinema_db`.
3. В файле `src/main/resources/application.properties` проверьте настройки подключения к БД:
   ```properties
   spring.datasource.url=jdbc:postgresql://localhost:5432/cinema_db
   spring.datasource.username={ваш логин}
   spring.datasource.password={ваш пароль}
   server.port=8081
   ```
4. Запустите проект через вашу IDE или с помощью Maven:
   ```bash
   mvn spring-boot:run
   ```
