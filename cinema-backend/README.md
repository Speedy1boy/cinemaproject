# Система бронирования билетов для кинотеатра (Cinema Booking System)

Серверная часть веб-приложения, предназначенная для просмотра актуального расписания фильмов, управления кинозалами, сеансами и бронирования мест в режиме реального времени. Проект реализован в рамках командной работы согласно техническому заданию.

Приложение построено по клиент-серверной архитектуре, где бэкенд предоставляет надежное REST API для взаимодействия с фронтенд-частью, обеспечивает централизованную обработку бизнес-логики, валидацию данных и разграничение прав доступа.

## Архитектура приложения

В проекте строго выдержана трехслойная архитектура (Layered Architecture) для обеспечения гибкости, масштабируемости и читаемости кода:

1. **Presentation Layer (Уровень представления):** REST-контроллеры (`@RestController`), принимающие HTTP-запросы от фронтенда, валидирующие входные DTO и возвращающие ответы в формате JSON.
2. **Business Logic Layer (Уровень бизнес-логики):** Сервисный слой (`@Service`), где сосредоточены все бизнес-правила (проверка доступности мест, недопущение повторного бронирования, расчет стоимости билетов).
3. **Data Access Layer (Уровень доступа к данным):** Репозитории (`@Repository`), использующие Spring Data JPA и Hibernate для эффективного выполнения CRUD-операций и взаимодействия с СУБД.

## Технологический стек

* Язык программирования: Java
* Фреймворк: Spring Boot 3.x (Spring Web, Spring Security)
* Доступ к данным: Spring Data JPA, Hibernate ORM
* База данных: PostgreSQL
* Сборщик проектов: Maven
* Документация API: Swagger / OpenAPI 3
* Тестирование: JUnit, Mockito

## Инструкция по локальному запуску бэкенда

### 1. Требования
* Java Development Kit (JDK) 25 или выше
* PostgreSQL СУБД
* Maven

### 2. Конфигурация базы данных
Перед запуском создайте пустую базу данных `cinema_db`. В файле `src/main/resources/application.properties` укажите локальные параметры доступа:
```properties
spring.datasource.url=jdbc:postgresql://localhost:5432/cinema_db
spring.datasource.username=your_postgres_user
spring.datasource.password=your_postgres_password
spring.jpa.hibernate.ddl-auto=update
